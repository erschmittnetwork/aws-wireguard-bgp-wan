Secure WAN, BGP, and AWS Automation

This site hosts a technical lab documenting the design, implementation, and validation of an AWS-hosted hub-and-spoke WAN architecture using WireGuard for encrypted transport, FRRouting (BGP) for dynamic routing, and AWS-native automation for high-availability failover.

The environment is implemented as a proof-of-concept while intentionally aligned with enterprise architectural principles: deterministic routing behavior, encrypted connectivity, automated failure handling, and operational observability. Although not intended to replace full SD-WAN platforms, the design reflects realistic constraints found in small-to-medium enterprise networks.

The solution demonstrates how a minimal cloud footprint—two hub nodes and multiple remote spokes—can deliver secure WAN connectivity, dynamic routing, and automated hub failover without reliance on proprietary controllers.

The author is a Cisco Certified Network Associate (CCNA) with a focus on routing, switching, and secure cloud-connected WAN architecture.

This site itself is delivered through a GitHub-to-AWS CI/CD pipeline. Source documentation and diagrams are version-controlled in GitHub. Automated workflows render Mermaid diagrams, build HTML and PDF artifacts using Pandoc, and publish content to AWS via S3 and CloudFront, enabling deterministic builds and continuous delivery of the portfolio.
