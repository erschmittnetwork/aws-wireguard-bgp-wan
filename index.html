This site hosts a technical lab demonstrating the design, implementation, and validation of an AWS-hosted hub-and-spoke WAN architecture using WireGuard for encrypted transport, FRRouting (BGP) for dynamic routing, and AWS-native automation for high-availability failover.

The environment is implemented as a proof-of-concept, but intentionally aligned with enterprise architectural principles: deterministic routing behavior, encrypted connectivity, automated failure handling, and operational observability. While the design does not attempt to replicate full SD-WAN platforms, all components reflect realistic operational constraints found in small-to-medium enterprise deployments.

The solution demonstrates how a minimal cloud footprint—two hub nodes and multiple remote spokes—can deliver encrypted WAN connectivity, dynamic routing, and automated hub failover without reliance on proprietary SD-WAN controllers.

This site itself is delivered through a GitHub-to-AWS CI/CD pipeline. Source documentation, diagrams, and infrastructure definitions are version-controlled in GitHub. On each update, automated workflows render Mermaid diagrams, build HTML and PDF artifacts using Pandoc, and publish the site to AWS via S3 and CloudFront. This ensures deterministic builds, reproducible documentation, and continuous delivery of the portfolio.
